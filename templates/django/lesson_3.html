Djangoda birinchi web dastur


Keling, amaliy mashg'ulotni boshlaylik.

Ushbu darsda foydalanuvchilar tomonidan so'rovlarga ovoz berish uchun mo'ljallangan web dasturini yaratish haqida gapiramiz.

U ikki qismdan iborat bo'ladi:

  • Foydalanuvchilar so'rovlarni ko'rib chiqishga va ularga ovoz berishga imkon beruvchi qismi (sayt).

  • So'rovlarni qo'shish, o'zgartirish va o'chirish imkonini beruvchi administrator paneli.

Django-ni allaqachon o'rnatgansiz deb taxmin qilamiz. Djangoning qaysi versiyani o'rnatilganini buyruq satrida(terminal ,bash console) quyidagi buyruqni bajarish orqali bilib olishingiz mumkin ($prefiksi ko'rsatiladi):

	

$ python -m django --version

Agar djangoni muvaffaqiyatli o'rnatgan bo'lsangiz unda django versiyasining raqami chiqadi.Aks xolda "Django degan modul topilmadi degan" xatolik ko'rasiz

Ko'rib chiqing

  • Pythonni o'rnatish
  • Pythonning qaysi versiyalari Django bilan ishlashi mumkin
  • Djangoni o'rnatish (rasmiy qo'llanma)

Loyixani boshlash

Terminal orqali loyixani saqlaydigan katalogga o'ting.Masalan:

	

$ cd C://user/UserName/desktop/djangoproject

Keyin django-admin buyrug'i yordamida loyixani boshlaymiz

	

$ django-admin startproject mysite

Bu djangoproject katalogi ichida mysite katalogini yaratadi .Agar muammo yuz bersa ushbu xavolani o'qing

Keling endi startproject buyrug'i natijasi bilan tanishsak:

mysite/
    manage.py
    mysite/
        __init__.py
        settings.py
        urls.py
        wsgi.py

Ushbu fayllar va kataloglar nima uchun kerakligi bilan tanishib chiqsak

  • mysite/ - loyixa(web dastur) uchun konteyner katalog.Katalog nomi Django uchun xech qanday axamiyatga ega emas siz uni xoxlagan nom bilan nomlashingiz mumkin

  • manage.py - fayli loyixa bilan xar xil xarakatlarni amalga oshirishimiz uchun utilita vazifasini bajaradi

  • mysite/ ichki katalogi loyihangizning Python moduli. Uning nomi ushbu moduldan biror narsani import qilish uchun ishlatiladi (masalan, mysite.urls).

  • mysite/__init__.py:Bo'sh fayl. Pythonga ushbu katalog Python paketi ekanligi xaqida xabar beradi . Agar Pythonni o'rganishni endi boshlagan bo'lsangiz, Python paketlari haqida ko'proq o'qing.

  • mysite/settings.py: Djangoning konfugratsiya va sozlamalari. settings.py xaqida ko'proq

  • mysite/urls.py Web dastur uchun urllar ko'rsatiladigan modul.Qisqa qilib aytganda web ilovani 'yurg`azishda' asosiy figuralardan biri

  • mysite/wsgi.py: WSGI uchun kirish nuqtasi.Web dasturni web serverga bog'lab ishga tushirish uchun kerak. WSGI xaqida ko'proq ma'lumotlarni bu yerdan topishingiz mumkin

Django "ishlab chiqarish" serveri

Django loyihamiz ishlayotganini tekshirib ko'raylik. Buning uchun terminal,buyruq satri oraqli mysite tashqi katalogiga o'ting va quyidagi buyruqlarni bajaring:

	

	$ python manage.py runserver

Buyruq satri yoki terminalingizda quyidagilarni ko'rishingiz mumkin

Performing system checks...

System check identified no issues (0 silenced).

You have unapplied migrations; your app may not work properly until they are applied.
Run 'python manage.py migrate' to apply them.

марта 27, 2020 - 15:50:53
Django version 3.0, using settings 'mysite.settings'
Starting development server at http://127.0.0.1:8000/
Quit the server with CONTROL-C.

Siz Django 'ishlab chiqarish' (razrabotka) serverini ishga tushirdingiz:Bu faqat Pythonda yozilgan yengil web-server. Biz uni Django-ga uladik, shunda siz tezda loyihani ishlab chiqishingiz mumkin, Apache (Open Server, XAMP, Denwer) kabi ishchi serverlarni o'rnatishga harakat qilmaysiz, boshingizni qotirmaysiz.

Endi ta'kidlash vaqti keldi: ushbu serverni 'jangovor' xar doim web dasturni "ko'tarib" turadigan server sifatida foydalanilmaydi. Bu faqat ishlab chiqishda (razrabotka vaqtida) foydalanish uchun mo'ljallangan. (Veb-serverlar emas, balki veb-ilovalar yaratish bilan shug'ullanyapmiz.)

Endi server ishga tushirilganda brauzerda quyidagi manzil bilan sahifani oching http://127.0.0.1:8000/ Siz "Congratulations" sahifasini ko'rishingiz kerak, 'raketasi' bilan. Hamma narsa xatoliksiz ishga tushdi!

Bu buyruq loyixa ichida polls deb nomlangan katalog (web-ilovani) xosil qiladi:

polls/

Djangoda birinchi taqdimotni(views) yozish

Keling, birinchi taqdimotni yozaylik (view). polls katalogini oching va views.py fayliga quyidagi Python kodini kiriting:

from django.http import HttpResponse


def index(request):
    
	return HttpResponse("Hello, world. You're at the polls index.")

Bu Djangoda eng oddiy taqdimot.Taqdimot (view) bizga brauzerda "Hello, world. You're at the polls index." yozuvini chiqarib ko'rsatadi.Buning uchun oldin taqdimotni URL yordamida 'chaqirishimiz' kerak va biz URLconf ga murojat qilamiz.

URLconf ishlatishiz uchun polls katalogida(web-ilovasida) urls.py nomli fayl yarating:

polls/
    __init__.py
    admin.py
    apps.py
    migrations/
        __init__.py
    models.py
    tests.py

    urls.py
    
    views.py

Xozirgina xosil qilgan faylga ushbu kodni qo'shing polls/urls.py:

polls/urls.py

	
from django.urls import path

from . import views

urlpatterns = [
    path('', views.index, name='index'),
]

Keyingi qadam,loyixaning bosh URLconfida polls web-ilovamizning urls ini ko'rsatishimiz.Ya'ni brauzerda mysite.com/polls manzili so'ralganda polls web ilovamizning URLconfi (urls.py) ishga tushsin. Buning uchun mysite/urls.py da django.urlsdan include funksiyasini import qilish kerak

mysite/urls.py

	
from django.contrib import admin
from django.urls import path, include



urlpatterns = [
    path('admin/', admin.site.urls),
    path('polls/',include("polls.urls")),
]

include() funksiyasi loyixamizdagi (web-ilovalar ichidagi) boshqa URLconf larga murojaat qilish imkonini beradi.

Include() asosiy vazifasi URL-larni qo'shish va ijro etishni osonlashtiradi.

Endi URLconf-ga indeks taqdimoti(views) ulandi.Server ishlayotganiga ishonch hosil qiling , agar server to'xtagan bo'lsa uni qaytadan ishga tushiring:

	
 $ python manage.py runserver

Brauzerda quyidagi manzilga o'ting http://localhost:8000/polls/, va ushbu matnni ko'rishimiz mumkin " Hello, world. You're at the polls index."

 	emas aksincha http://localhost:8000/polls/ manzilini so'rayotganingizni tekshiring

path() funksiyasi ikkita majburiy: route , view va ikkita ixtiyoriy: kwargs, name argumentlar jo'natadi.Quyida ushbu argumentlar nima uchun kerakligini ko'rib chiqamiz

path(): route argumenti

route-URL manzilni o'z ichiga olgan satr.Url so'rovni(masalan mysite.com/polls) qayta ishlashda Django urlpatterns-dagi birinchi shablondan qidirishni boshlaydi va kerakli URLga mos keladigan biror narsani topguniga qadar har bir url manzillarni so'rov bilan taqqoslab, ro'yxatni pastga tushiradi.

Url manzillar GET va POST parametrlarini yoki domen nomi qidiruvini amalga oshirmaydi. Misol uchun,https://www.example.com/myapp/, so'rovida URLconf myApp/ web-ilovasidagi url manzillarni qidirishni boshlaydi. So'rov https://www.example.com/myapp/?Page=3. ga o'zgargan taqdirda xam URLconf myapp/ ni qidiradi.

path(): view argumenti

Django tegishli shablonni topgach, HttpRequest obyekti qaytaradigan url manzil uchun biriktrilgan taqdimot(views) funktsiyasini chaqiradi va url manzildan "qo'lga kiritilgan" qiymatni birinchi argument sifatida uzatadi.Bu masalaga keyinroq batafsil to'xtalamiz.

path(): kwargs argumenti

Python lug'ati (dict) yordamida taqdimotga(views) uzatiladigan qo'shimcha argumentlar.Darsimizning keyingi qismlarida bu xususiyatidan foydalanamiz.

path(): name argumenti

URL-manzil uchun o'ziga xos nom berish, Django loyixamizning boshqa 'joylaridan' ayniqsa shablonlardan berilgan nom orqali URl-manzilimizga murojaat qilish imkonini beradi. Ushbu kuchli funksiya sizni loyihangizning URL shablonlariga faqat bitta faylga o'zgartirish kiritib global o'zgarishlarni amalga oshirish imkonini beradi.

Url so'rovlar va javoblar(views) asoslari xaqida boshlang'ich ma'lumotlarga ega bo'lgan bo'lsangiz, ma'lumotlar ombori bilan ishlashni boshlash uchun ushbu qo'llanmaning ikkinchi qismini o'qing.
