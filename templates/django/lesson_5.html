Djangoda birinchi web dastur 3-qism

Ushbu qo'llanma sizni Django bilan tanishuvni ikkinchi qismida to'xtalgan joydan tanishtirishda davom etadi. Biz Web-poll ilovasini davom ettiramiz va ochiq interfeysni yaratishga e'tibor qaratamiz - "ko'rinishlar".

Tez ko'rib chiqish

Taqdimot, odatda, ma'lum bir funktsiyani bajaradigan va ma'lum bir shablonga ega bo'lgan Django ilovasidagi veb-sahifaning "turi" dir. Misol uchun, blog ilovasida quyidagi fikrlar bo'lishi mumkin:

  • Blogning asosiy sahifasi-eng so'nggi yozuvlarni ko'rsatadi.
  • "Detalization" sahifasi-bitta yozuvning alohida sahifasi.
  • Yilga asoslangan arxiv sahifasi - bu yilgi yozuvlar bilan barcha oylarni ko'rsatadi
  • Bir oyga asoslangan arxiv sahifasi-barcha kunlarni ushbu oyda yozuvlar bilan ko'rsatadi.
  • Kunlik arxiv sahifasi-ma'lum bir kun uchun barcha yozuvlarni ko'rsatadi.
  • Izohlar bilan qilingan harakatlar-ushbu yozuvga sharhlar joylashtirishni ko'rib chiqadi.

Bizning so'rov dasturimizda quyidagi to'rtta taqdimot bo'ladi:

  • Savollarning asosiy sahifasi-so'nggi bir nechta savollarni aks ettiradi.
  • Savol sahifasi-savolning matnini natijasiz, ammo ovoz berish shakli bilan ko'rsatadi.
  • Savol natijalari sahifasi-muayyan savol uchun natijalarni ko'rsatadi.
  • Ovoz berish-muayyan masala bo'yicha muayyan tanlov uchun ovoz beradi.

Django-da veb-sahifalar va boshqa kontent ko'rinishlarga ko'ra yetkaziladi. Har bir vakillik Python funktsiyasi bilan ifodalanadi (yoki sinflarga asoslangan ko'rinishda usul). Django so'ralgan URLni (aniqrog'i, domen nomidan keyin URLning bir qismini) o'rganib, ko'rinishni tanlaydi.

Endi internetda bo'lish "ME2/Sites/dirmod" kabi go'zallikka duch kelishi mumkin.asp?sid=&type=gen&mod=Core+Pages&gid=A6CD4967199A42D9B65B1B». Django sizga juda oqlangan URL shablonlarini yaratishga imkon beradi.

URL shabloni umumiy URL shaklidir, masalan:/News archive///.

URLdan taqdimotga o'tish uchun Django URLconfs deb ataladi. URLconf URL shablonlarini taqdimotlar bilan mos keladi.

Ushbu ta'lim URLconfs foydalanish bo'yicha asosiy ko'rsatmalarni o'z ichiga oladi, va siz uchun murojaat qilishingiz mumkin URL dispatcher qo'shimcha ma'lumot olish uchun.

Loyixamiz uchun taqdimot (views) yozamiz

Keling, bir nechta fikrlarni qo'shamiz polls/views.py. ushbu fikrlar bir-biridan farq qiladi, chunki ular argumentni qabul qilishadi:

	

polls/views.py¶

def detail(request, question_id):
    return HttpResponse("You're looking at question %s." % question_id)

def results(request, question_id):
    response = "You're looking at the results of question %s."
    return HttpResponse(response % question_id)

def vote(request, question_id):
    return HttpResponse("You're voting on question %s." % question_id)

Ushbu yangi ko'rinishlarni polls moduliga ulang.quyidagi yo'l qo'shib URL () qo'ng'iroqlar:

	

polls/urls.py¶

from django.urls import path

from . import views

urlpatterns = [
    # ex: /polls/
    path('', views.index, name='index'),
    # ex: /polls/5/
    path('/', views.detail, name='detail'),
    # ex: /polls/5/results/
    path('/results/', views.results, name='results'),
    # ex: /polls/5/vote/
    path('/vote/', views.vote, name='vote'),
]
